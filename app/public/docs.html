Введение в API

Добро пожаловать в документацию нашего API. Этот API предоставляет возможность взаимодействия с нашей платформой программно.

Базовый URL для всех запросов: https://api.example.com/v1

Все ответы возвращаются в формате JSON.

Аутентификация

Для доступа к API необходимо использовать API-ключ в заголовках запросов.

POST /auth/login

Параметры запроса

Параметр Тип Описание
email String Email пользователя
password String Пароль пользователя

Пример запроса

curl -X POST https://api.example.com/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "yourpassword"
  }'

Пример ответа

{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 123,
    "name": "John Doe",
    "email": "user@example.com"
  }
}

Пользователи

API для управления пользователями системы.

GET /users

Получить список всех пользователей. Требует аутентификации.

Пример ответа

{
  "users": [
    {
      "id": 1,
      "name": "John Doe",
      "email": "john@example.com"
    },
    {
      "id": 2,
      "name": "Jane Smith",
      "email": "jane@example.com"
    }
  ],
  "total": 2
}

Посты

API для работы с постами в системе.

POST /posts

Создать новый пост. Требует аутентификации.

Параметры запроса

Параметр Тип Описание
title String Заголовок поста
content String Содержание поста

Ошибки

API использует стандартные HTTP коды статусов для указания на успех или неудачу запроса.

Код Значение
200 OK - Запрос выполнен успешно
201 Created - Ресурс успешно создан
400 Bad Request - Неверный запрос
401 Unauthorized - Требуется аутентификация
404 Not Found - Ресурс не найден
500 Internal Server Error - Ошибка сервера
